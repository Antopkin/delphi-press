# Delphi Press

**Delphi Press** — это автоматизированная система прогнозирования заголовков средств массовой информации на заданную целевую дату. Система анализирует текущее состояние геополитических, экономических и медийных контекстов через специализированный мультиагентный пайплайн и генерирует ранжированные предсказания заголовков с уровнями уверенности и обоснованиями.

Исследователи-политологи и аналитики медиа сталкиваются с задачей предсказания того, на какие события обратит внимание конкретное издание в конкретный день. Это требует анализа информационных потоков, понимания редакционной позиции, моделирования вероятности события и генерации заголовков в стиле целевого СМИ. Ручной анализ требует дней работы. **Delphi Press** автоматизирует весь процесс через специализированный подход ensemble forecasting — мультиагентный конвейер, вдохновлённый методом Дельфи.

## Ключевые цифры

- **9 стадий**: Последовательно-параллельный гибридный пайплайн
- **18 агентов**: Специализированные роли (коллекторы, аналитики, прогнозисты, генераторы)
- **28 LLM-задач**: От сбора данных до финальной гарантии качества
- **5 экспертных персон**: Реалист, Геостратег, Экономист, Медиа-эксперт, Адвокат дьявола
- **2 раунда консенсуса**: Дельфи R1 и R2 для устойчивого прогнозирования

## Поддерживаемые режимы развёртывания

### Web UI
Веб-интерфейс с поддержкой:
- Собственный API-ключ пользователя (OpenRouter)
- Стоимость: **$0,10–15 за прогноз**
- REST API + поддержка расписания
- Сохранение в БД и аналитика

### Claude Code
CLI-предсказание через скилл Claude Code:
- Без API-ключа (включено в подписку Code Max)
- Вывод в Markdown-отчёт
- Полная интеграция с субагентами Claude

## Технический стек

- **Язык**: Python 3.12+
- **Backend**: FastAPI + Uvicorn
- **Очередь задач**: ARQ (Redis)
- **База данных**: SQLite + aiosqlite
- **Валидация**: Pydantic v2
- **Frontend**: Tailwind CSS 4.2.2 + Vanilla JS
- **LLM**: OpenRouter (200+ моделей)
- **Развёртывание**: 4-контейнерный Docker Compose

## Что система делает

1. **Сбор данных (стадия 1)**: Сбор сигналов из RSS, запланированных событий, истории издания и рынков предсказаний
2. **Анализ (стадии 2–3)**: Кластеризация событий, моделирование траекторий, анализ геополитического/экономического/медийного контекстов
3. **Консенсус Дельфи (стадии 4–5)**: Два раунда экспертного прогнозирования с медиацией
4. **Синтез (стадии 6–7)**: Консенсус судьи, анализ подачи материала
5. **Генерация (стадия 8)**: Генерация стилистически подходящих заголовков
6. **Гарантия качества (стадия 9)**: Факт-чек и финальные прогнозы

Каждая стадия поддерживает параллелизм, с плавной деградацией через логику `min_successful`.
