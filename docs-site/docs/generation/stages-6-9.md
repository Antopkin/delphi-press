# Стадии 6-9: Агрегация, фрейминг, генерация и контроль качества

Заключительные четыре стадии пайплайна преобразуют персональные прогнозы в финальные заголовки.

---

## Стадия 6: Judge — агрегация и ранжирование

Стадия реализует финальную агрегацию персональных прогнозов *Delphi ensemble* и выбор событий для генерации заголовков. Двухшаговый алгоритм.

### Шаг 6a: Агрегация на уровне событий

#### Парсинг оценок

Агент Judge принимает результаты раунда 2 либо раунда 1 в качестве fallback. Каждая оценка содержит список прогнозов конкретной персоны с вероятностями.

#### Взвешенная медиана

Для каждого события (идентифицируемого `event_thread_id`) агент вычисляет взвешенную медиану вероятностей пяти персон:

| Персона | Базовый вес |
|---------|-------------|
| **REALIST** | 0.22 |
| **GEOSTRATEG** | 0.20 |
| **ECONOMIST** | 0.20 |
| **MEDIA_EXPERT** | 0.18 |
| **DEVILS_ADVOCATE** | 0.20 |

#### Horizon-адаптивные коррекции

Веса корректируются в зависимости от временного горизонта (расстояния до целевой даты):

| Горизонт | REALIST | GEO | ECON | MEDIA | DEVIL |
|----------|---------|-----|------|-------|-------|
| **Immediate** (0–2 дня) | −0.02 | −0.02 | +0.02 | +0.03 | −0.01 |
| **Near** (3–4 дня) | 0.00 | 0.00 | 0.00 | 0.00 | +0.02 |
| **Medium** (5+ дней) | +0.03 | +0.02 | −0.01 | −0.02 | −0.02 |

**Обоснование:** для немедленных прогнозов важны циклы новостей (Media Expert) и календарные факторы (Economist); для дальних горизонтов — структурные тренды (Realist, Geostrateg).

#### Инъекция рыночного сигнала

Если в контексте пайплайна присутствуют *foresight signals* (Polymarket), агент выполняет нечёткое сопоставление события с рынками через метрику Jaccard:

- **Tier 1:** точное совпадение (similarity ≥ 0.65)
- **Tier 2:** частичное совпадение (similarity ≥ 0.40 и Jaccard ≥ 0.3)

Если совпадение найдено, добавляется псевдо-персона «Market» с динамическим весом:

$$w_{\text{market}} = 0.15 \times \max(0.5, \min(\log_{10}(V)/6, 1.5)) \times \frac{1}{1 + \text{vol}} \times \text{reliable} \times r_{\text{horizon}}$$

где $V$ — объём рынка, $r_{\text{horizon}}$ учитывает дни до закрытия рынка, $\text{reliable} = 0$ если распределение ненадёжно, иначе $1$.

!!! note "Пороги инъекции"
    Market-сигнал инъектируется только при $p_{\text{market}} \geq 0.10$ (**MARKET_MIN_PROBABILITY**) для исключения шума от неликвидных рынков. Дополнительно применяется фильтр ликвидности: рынки с объёмом < **$10,000** (**MARKET_MIN_LIQUIDITY**) исключаются при построении индекса.

#### Калибровка: Platt Scaling с экстремизацией

После вычисления взвешенной медианы $p_{\text{raw}}$ применяется *Platt scaling* с параметром экстремизации $a = 1.5$:

$$p_{\text{calibrated}} = \sigma(a \cdot \text{logit}(p_{\text{raw}}) + b)$$

где $\text{logit}(p) = \ln(p/(1-p))$, $\sigma(x) = 1/(1+e^{-x})$, $b = 0.0$.

Экстремизация ($a > 1$) усиливает уверенные прогнозы и ослабляет неуверенные, повышая калибровку на реальных данных.

#### Уровни согласия

Разброс вероятностей $\text{spread} = \max(p_i) - \min(p_i)$ классифицируется как:

- **CONSENSUS:** $\text{spread} < 0.15$ (тесное согласие)
- **MAJORITY_WITH_DISSENT:** $0.15 \leq \text{spread} < 0.30$
- **CONTESTED:** $\text{spread} \geq 0.30$ (глубокие разногласия)

При CONTESTED применяется штраф неопределённости ($\times 0.8$) к сырой вероятности.

#### Агрегация временных полей

Для каждого события агрегируются поля:
- `predicted_date` — медиана по датам персон
- `uncertainty_days` — среднее
- `scenario_types` — объединение
- `causal_dependencies` — объединение

Выход: `PredictedTimeline` с отсортированными по дате `TimelineEntry`.

### Шаг 6b: Отбор заголовков

#### Оценка новостности

Для каждого события в timeline вычисляется headline score:

$$\text{score} = p_{\text{calib}} \times n \times (1 - s) \times r$$

где:
- $p_{\text{calib}}$ — калиброванная вероятность события
- $n$ — новостная ценность (среднее по персонам)
- $s$ — коэффициент насыщенности (текущее заполнение 0.0)
- $r$ — релевантность для издания (по умолчанию 1.0)

#### Временная близость

Коэффициент temporal proximity штрафует события, далёкие от целевой даты:

$$\text{temporal\_factor} = \max(0.5, 1.0 - 0.05 \cdot \text{days\_away})$$

#### Топ-7 + дикие карты

Отбираются 7 событий с наивысшим финальным score. Дополнительно добавляются до 2 «дикие карты» (black swan сценарии от Devil's Advocate) с новостной ценностью ≥ 0.7.

Вывод: `RankedPrediction[]` с полями `headline_score`, `rank`, `is_wild_card`.

## Стадия 7: FramingAnalyzer — редакционный угол

Стадия анализирует, *как конкретное издание подаст событие*. Один и тот же факт получает разные редакционные углы в зависимости от политики и стиля издания.

### Девять стратегий фрейминга

| Стратегия | Пример |
|-----------|---------|
| **THREAT** | Угроза для здоровья, безопасности, экономики |
| **OPPORTUNITY** | Шанс, перспектива развития |
| **CRISIS** | Острая проблема, обострение ситуации |
| **ROUTINE** | Регулярное, периодическое событие |
| **SENSATION** | Шокирующие подробности, неожиданный поворот |
| **ANALYTICAL** | Глубокий анализ, контекст, причины |
| **HUMAN_INTEREST** | История людей, личные драмы |
| **NEUTRAL_REPORT** | Фактический отчёт без оценки |
| **CONFLICT** | Столкновение позиций, дебаты |

### Структура FramingBrief

Для каждого `RankedPrediction` генерируется `FramingBrief` с полями:

- **framing_strategy** — одна из девяти стратегий
- **angle** — конкретный угол подачи (1–2 предложения)
- **emphasis_points** — 2–5 элементов, которые издание подчеркнёт
- **omission_points** — 0–5 элементов, которые издание приглушит или опустит
- **headline_tone** — тревожный / нейтральный / оптимистичный / ироничный
- **likely_sources** — 1–5 источников, на которые сошлётся издание
- **section** — раздел издания (Политика, Экономика, Главное и т.д.)
- **news_cycle_hook** — привязка к текущему новостному циклу
- **editorial_alignment_score** — насколько событие соответствует редакционной линии (0 — совсем не в профиле, 1 — идеально)

### Входные данные

Для анализа используется `OutletProfile` с:
- Редакционной позицией (editorial_stance)
- Типичными разделами (sections)
- 10–20 примерами недавних заголовков (sample_headlines)
- Тональным профилем (tone_profile: нейтральный, сенсационный, аналитический)
- Типичными источниками (sourcing_patterns)

**Модель:** Claude Sonnet (OpenRouter) с temperature 0.5 для консервативного reasoning.

## Стадия 8: StyleReplicator — генерация заголовков

Стадия генерирует 2–3 варианта заголовка и первого абзаца в стиле целевого издания.

### Few-shot инъекция стиля

StyleReplicator получает из `OutletProfile`:

- 10–20 реальных заголовков издания (примеры для имитации)
- 3–5 примеров первых абзацев (лидов)
- Метрики стиля:
  - `avg_headline_length` — средняя длина в символах
  - `headline_case` — нижний / верхний / sentence case / title case
  - `tone_profile` — нейтральный / острый / сенсационный / аналитический
  - `quotes_usage` — частота использования кавычек
  - `colon_usage` — частота двоеточия
  - `avg_lede_length` — средняя длина первого абзаца в словах

### Мультиязычность

Выбор модели зависит от языка издания (все языки используют Claude Sonnet через OpenRouter).

### Промпт структура

Системный промпт содержит идентичность («Ты — автор заголовков для издания X»), параметры стиля и примеры. Пользовательский промпт:

- Событие и его вероятность
- FramingBrief (стратегия, угол, тон, источники)
- Запрос: сгенерировать 2–3 варианта с первым абзацем

**Температура:** 0.8 для творческого разнообразия.

**Выход:** `GeneratedHeadline[]` с полями `headline`, `first_paragraph`, `headline_language`, `length_deviation`.

## Стадия 9: QualityGate — контроль качества

Финальный фильтр с тремя независимыми проверками для каждого заголовка: **PASS** / **REJECT** / **REVISE** / **DEPRIORITIZE** / **MERGE**.

### Проверка 1: Фактическая правдоподобность

#### Задача

Оценить внутреннюю непротиворечивость заголовка и первого абзаца, соответствие известным фактам. **НЕ** оценивает, произойдёт ли событие.

#### Критерии

- Нет ли противоречий с общеизвестными фактами?
- Корректны ли упомянутые даты, должности, названия?
- Логически непротиворечив ли сценарий?
- Не является ли событие физически / логически невозможным?
- Не устарела ли предпосылка?

#### Оценка (1–5)

| Балл | Интерпретация |
|------|----------------|
| 5 | Полностью правдоподобно, фактических противоречий не найдено |
| 4 | Мелкие неточности, не влияющие на смысл |
| 3 | Спорные утверждения, но принципиально не ошибочные |
| 2 | Фактическая ошибка или серьёзное логическое противоречие |
| 1 | Явно абсурдный или невозможный сценарий |

**Порог:** $\text{factual\_score} < 3$ → **REJECT** (проблема в прогнозе, а не в генерации).

**Модель:** Claude Sonnet, temperature 0.2 (консервативно).

### Проверка 2: Стилистическая аутентичность

#### Задача

Оценить, насколько заголовок и первый абзац стилистически соответствуют целевому изданию.

#### Критерии

- Типична ли лексика для этого издания?
- Соответствуют ли синтаксические конструкции?
- Подходит ли длина заголовка?
- Верный ли тон?
- «Smell test»: похоже ли это на реальную публикацию этого издания?

#### Оценка (1–5)

| Балл | Интерпретация |
|------|----------------|
| 5 | Неотличимо от настоящей публикации |
| 4 | Стиль в целом верный, мелкие шероховатости |
| 3 | Узнаваемый стиль, но с заметными отклонениями |
| 2 | Стиль не соответствует изданию |
| 1 | Совершенно не похоже на это издание |

**Порог:** $\text{style\_score} < 3$ → **REVISE** (одна попытка ревизии в StyleReplicator).

**Модель:** Claude Sonnet (для всех языков), temperature 0.2.

### Проверка 3: Дедупликация

#### Внутренняя дедупликация

Вычисляются embeddings всех заголовков (text-embedding-3-small через OpenRouter). Если cosine similarity между двумя заголовками > 0.85, слабее по среднему score помечается дубликатом.

#### Внешняя дедупликация

Если cosine similarity заголовка с реальными заголовками из `OutletProfile.sample_headlines` > 0.80, заголовок помечается как external duplicate.

### Матрица решений

| factual | style | dup | Решение |
|---------|-------|-----|----------|
| ≥ 3 | ≥ 3 | нет | **PASS** |
| < 3 | любой | любой | **REJECT** |
| ≥ 3 | < 3 | нет | **REVISE** |
| ≥ 3 | ≥ 3 | внутр. | **MERGE** |
| ≥ 3 | ≥ 3 | внеш. | **DEPRIORITIZE** |

### Обработка решений

- **PASS:** заголовок добавляется в `FinalPrediction`
- **REJECT:** заголовок отклоняется (без повторных попыток)
- **REVISE:** заголовок отправляется обратно в StyleReplicator с обратной связью (максимум 1 попытка), затем перепроверяется стиль
- **MERGE:** дубликат не добавляется, более сильный вариант уже в списке
- **DEPRIORITIZE:** заголовок добавляется в конец списка

### Выход

На входе: 14–21 вариант заголовков (2–3 на каждое из 7 событий).
На выходе: 5–7 `FinalPrediction` после фильтрации.

Каждый `FinalPrediction` содержит:

- Основной заголовок и первый абзац
- 1–2 альтернативных заголовка
- Калиброванную вероятность события
- Уровень согласия экспертов
- Стратегию фрейминга
- Язык заголовка
- Флаг wild card

!!! warning "Параллельная обработка"
    До 5 заголовков проверяются одновременно с timeout 180 сек. При timeout применяется нейтральная оценка (score = 3, пропускает заголовок дальше).
