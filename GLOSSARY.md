# Глоссарий Delphi Press

Доменные термины проекта. Используй эти определения при работе с кодом и спеками.

## Метод Дельфи

| Русский | English (в коде) | Определение |
|---|---|---|
| Метод Дельфи | Delphi Method | Структурированная групповая экспертная оценка: 2 раунда с анонимной обратной связью (Dalkey & Helmer, 1963). В проекте: 5 LLM-агентов как эксперты. |
| Раунд 1 | Delphi R1 | Первая итерация: 5 персон независимо оценивают траектории событий. Каждая персона использует свою LLM-модель. Анонимность: агент НЕ видит чужие оценки. |
| Раунд 2 | Delphi R2 | Вторая итерация: персоны пересматривают оценки после получения синтеза медиатора. Анонимность сохраняется (эксперты помечены A, B, C). |
| Медиация | Mediation | **НЕ юридическая медиация.** Стадия между R1 и R2: нейтральный агент-медиатор структурирует расхождения, формулирует фактологические вопросы для персон. |
| Синтез медиатора | Mediator Synthesis | Выход медиатора: зоны консенсуса (spread < 0.15), ключевые вопросы по расхождениям (spread ≥ 0.15), пробелы в анализе, каскадные зависимости. |
| Консенсус | Consensus | Согласие экспертов: разброс оценок < 0.15. Не требует пересмотра в R2. |
| Расхождение | Disagreement | Разброс оценок ≥ 0.15. Медиатор формулирует конкретные вопросы для уточнения. |
| Взвешенная медиана | Weighted Median | Метод агрегации вероятностей с весами по историческому Brier Score персон. Не простое среднее. |
| Калибровка | Calibration | Корректировка сырых вероятностей LLM (Platt scaling + экстремизация) для устранения систематических смещений. |

## Стадии пайплайна

| # | Русский | English | Агенты | Вход → Выход |
|---|---|---|---|---|
| 1 | Сбор данных | Data Collection | news_scout, event_calendar, outlet_historian | — → SignalRecord[], ScheduledEvent[], OutletProfile |
| 2 | Идентификация событий | Event Identification | event_trend_analyzer | Signals → EventThread[] (~20 кластеров) |
| 3 | Анализ траектории | Trajectory Analysis | geopolitical, economic, media analysts | Threads → EventTrajectory[], CrossImpactMatrix |
| 4 | Дельфи R1 | Delphi Round 1 | 5 персон (параллельно) | Trajectories → PersonaAssessment[] |
| 5 | Дельфи R2 | Delphi Round 2 | mediator → 5 персон | Assessments → MediatorSynthesis → RevisedAssessment[] |
| 6 | Консенсус | Consensus & Selection | judge | Assessments → RankedPrediction[] (top-7 + wild cards) |
| 7 | Фрейминг | Framing Analysis | framing_analyzer | Predictions + OutletProfile → FramingBrief[] |
| 8 | Генерация | Style-Conditioned Generation | style_replicator | FramingBriefs → GeneratedHeadline[] (2-3 варианта) |
| 9 | Контроль качества | Quality Gate | quality_gate | Headlines → FinalPrediction[] (scores 1-5) |

## Данные и анализ

| Русский | English (в коде) | Определение |
|---|---|---|
| Сигнал | SignalRecord | Атомарная единица информации: новость с заголовком, URL, источником, датой, языком, категориями, оценкой релевантности. Собирается 100-200 штук. |
| Событийная нить | EventThread | Кластер связанных сигналов, объединённых общей темой. Одна нить = одна потенциальная новость. **Не путать с Signal** — много сигналов → одна нить. |
| Траектория события | EventTrajectory | Прогноз развития события: текущее состояние, импульс, 3-4 сценария с вероятностями. |
| Матрица перекрёстных влияний | CrossImpactMatrix | Зависимости между событиями: "если A произойдёт, как изменится вероятность B?" |
| Профиль издания | OutletProfile | Характеристика СМИ: редакционная позиция, темы, аудитория, тон, примеры заголовков. Кешируется 30 дней. |

## Прогнозирование

| Русский | English (в коде) | Определение |
|---|---|---|
| Вероятность события | Event Probability | Оценка 0.0-1.0: произойдёт ли событие к target_date. **Не путать с новостной ценностью** — это разные вопросы. |
| Новостная ценность | Newsworthiness | Оценка 0.0-1.0: будет ли издание освещать событие. Зависит от outlet, не только от события. |
| Ранжированный прогноз | RankedPrediction | Выход судьи: событие + калиброванная вероятность + новостная ценность + agreement_level + reasoning. |
| Дикий карт | Wild Card | Прогноз от адвоката дьявола вне top-7, но с newsworthiness > 0.7. Макс. 2 шт. в финальных результатах. |
| Уровень согласия | Agreement Level | consensus (spread < 0.15), contested (spread > 0.30), majority with dissent (0.15-0.30). |

## Генерация и качество

| Русский | English (в коде) | Определение |
|---|---|---|
| Фрейминг | Framing | Редакционный угол подачи: threat, opportunity, crisis, routine, sensation, analysis. **Не исследовательский bias.** |
| Фрейминг-бриф | FramingBrief | Инструкция для генератора: угол, тон, акценты, что опустить, стратегия источников. |
| Стилевой профиль | Style Profile | Характеристики стиля издания: длина заголовков, лексика, пунктуация, жанровые конвенции. |
| Факт-чек | Fact Check | Стадия 9a: проверка на противоречия с известными фактами. Оценка 1-5. Модель: Claude Sonnet. |
| Стилевая проверка | Style Check | Стадия 9b: соответствие стилю издания. Оценка 1-5. Модель: YandexGPT. |

## Техническая инфраструктура

| Русский | English (в коде) | Определение |
|---|---|---|
| Контекст пайплайна | PipelineContext | Мутабельный контейнер состояния, передаваемый через 9 стадий. Агенты читают входные слоты, пишут выходные. |
| Результат агента | AgentResult | Frozen dataclass: success, data (dict), duration_ms, tokens_in/out, cost_usd, error. |
| Оркестратор | Orchestrator | Координатор 9 стадий: последовательность стадий, параллельность агентов, fail-soft (min_successful). |
| Реестр агентов | AgentRegistry | DI-контейнер: регистрирует экземпляры агентов и LLM-клиент. |
| Роутер моделей | ModelRouter | Маршрутизация task_id → модель с fallback-цепочкой и контролем бюджета. |
| Модельное разнообразие | Model Diversity | Стратегия использования разных LLM-семейств для максимизации дисперсии ошибок ансамбля. |
| Brier Score | Brier Score | Метрика калибровки (0-1): среднеквадратичная ошибка прогнозов. 0.0 = идеал, 0.25 = случайное угадывание. |

## Критические разграничения

| Риск путаницы | Правильное определение |
|---|---|
| Медиация (юридическая) vs Медиация (Дельфи) | В Delphi Press: структурирование экспертных расхождений, НЕ разрешение юридических споров. |
| Вероятность события vs Новостная ценность | Probability = "произойдёт ли X?". Newsworthiness = "напишет ли издание Y об X?". Разные вопросы, объединяются в финальном скоринге. |
| Сигнал vs Событийная нить | Signal = одна новость. EventThread = кластер сигналов по теме. Много сигналов → одна нить. |
| Калибровка vs Согласие | Calibration = корректировка распределений вероятностей. Agreement = разброс мнений экспертов. |
| Фрейминг (редакционный) vs Фрейминг (bias) | Редакционный угол подачи (threat/opportunity), НЕ исследование предвзятости в смысле media studies. |
