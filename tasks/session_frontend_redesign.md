# Задача: Редизайн страниц прогресса и результатов

## Контекст

Главная страница (`/`) имеет отличный дизайн: Newsreader/Source Sans 3, чистая типографика, OKLCH цвета, хорошее белое пространство. Две другие страницы отстают по качеству layout и типографики:

1. **Страница прогресса** (`/predict/{id}`) — показывает pipeline из 9 стадий (сбор новостей → эксперты → генерация). Сейчас это простой вертикальный список с кружочками и прогресс-баром на 0%. Проблема: layout и типографика слабые, не соответствуют уровню главной.

2. **Страница результатов** (`/results/{id}`) — показывает итоговые прогнозы с заголовками, вероятностями, reasoning, market signal. Функционально работает, но визуально перегружена, layout и типографика не на уровне. Много UI-элементов (кнопки, панели, карточки), нужно навести порядок.

## Что нужно

Привести обе страницы к уровню качества главной. Фокус на:
- **Layout** — пространство, иерархия, ритм
- **Типографика** — шрифты, размеры, веса, отступы
- **Визуальная согласованность** с главной страницей

Пользователь открыт к разным вариантам. Нужно:
1. Провести аудит текущего состояния обеих страниц
2. Предложить 2-3 варианта редизайна для каждой
3. Задать уточняющие вопросы пользователю
4. Реализовать выбранный вариант

## Скиллы Impeccable

Используй максимально. Рекомендуемый порядок:
1. `/critique` — аудит текущего дизайна обеих страниц
2. `/arrange` — layout, spacing, visual rhythm
3. `/typeset` — типографика, иерархия шрифтов
4. `/distill` — убрать лишнее, упростить
5. `/polish` — финальная доводка
6. `/adapt` — responsive check

Дополнительно по необходимости: `/colorize`, `/animate`, `/delight`, `/normalize`.
